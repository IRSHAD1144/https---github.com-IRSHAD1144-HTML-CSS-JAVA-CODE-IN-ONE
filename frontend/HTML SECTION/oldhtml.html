OBSOLETE TAGS (REMOVED)

These tags are removed from modern HTML. Browsers may still support them, but DO NOT USE in new websites.

Added: 1991 | Removed: HTML5

<plaintext>


Why removed: Security risk and poor design. Everything after it became unrenderable HTML.

Old way:

<plaintext>This is just text <b>bold won't work</b>
Added: 1991 | Removed: HTML5

<listing>


Why removed: Redundant, replaced by <pre> and CSS.

Old way:

<listing> indented text </listing>
Added: 1991 | Removed: HTML5

<xmp>


Why removed: Treated everything as literal text, broke page parsing. Use <pre><code> instead.

Old way:

<xmp>This <b>bold</b> will not be bold.</xmp>
Added: 1991 | Removed: HTML5

<isindex>


Why removed: Obsolete search input. Use <input type="search">.

Old way:

<isindex>
Added: 1991 | Removed: HTML5

<nextid>


Why removed: NeXTSTEP editor feature, never standard.

Old way:

<nextid n=27>
Added: HTML5 | Removed: HTML5.1

<keygen>


Why removed: Security and complexity. Use Web Crypto API.

Old way:

<keygen name="security">
Added: 1990s | Removed: HTML5

<spacer>


Why removed: Presentational, CSS margin/padding replaces it.

Old way:

<spacer type="horizontal" size="50">
Added: IE only | Removed: HTML5

<bgsound>


Why removed: Autoplay background sound, user hostile. Use <audio> with user gesture.

Old way:

<bgsound src="music.mid" loop="infinite">
Added: IE only | Removed: HTML5

<marquee>


Why removed: Annoying scrolling text, never standard. Use CSS animations sparingly.

Old way:

<marquee>Scrolling text</marquee>
Added: Netscape | Removed: HTML5

<blink>


Why removed: The most hated tag, accessibility nightmare.

Old way:

<blink>Flashing text</blink>
Added: HTML4 | Removed: HTML5

<frame>


Why removed: Broke bookmarks and SEO. Use CSS Grid/Flexbox.

Old way:

<frameset cols="25%,75%"><frame src="menu.html"></frameset>
Added: HTML4 | Removed: HTML5

<frameset>


Why removed: Replaced <body>, same issues as frames.

Old way:

<frameset rows="50%,50%">...</frameset>
Added: HTML4 | Removed: HTML5

<noframes>


Why removed: Fallback for frames – frames are gone.

Old way:

<noframes>Your browser does not support frames.</noframes>
Added: HTML3.2 | Removed: HTML5

<applet>


Why removed: Java plugin security risks. Use JavaScript/WebAssembly.

Old way:

<applet code="MyApplet.class" width="200" height="200"></applet>
Added: Netscape/IE | Removed: HTML5 (redefined)

<embed> (old version)


Why removed: Plugin-specific, replaced by modern <embed>, <video>, <audio>.

Old way:

<embed src="movie.swf">
Added: Netscape/IE | Removed: HTML5

<noembed>


Why removed: Fallback for obsolete plugins.

Old way:

<noembed>Need Flash</noembed>
Added: HTML4 | Removed: HTML5

<acronym>


Why removed: Replaced by <abbr> (covers both acronyms and abbreviations).

Old way:

<acronym title="World Wide Web">WWW</acronym>
Added: IE/early | Removed: HTML5

<basefont>


Why removed: Presentational, use CSS on <body>.

Old way:

<basefont face="Arial" size="3">
Added: HTML3.2 | Removed: HTML5

<big>


Why removed: Presentational, use CSS font-size.

Old way:

This is <big>bigger text</big>.
Added: HTML3.2 | Removed: HTML5

<center>


Why removed: CSS text-align: center or margin: auto.

Old way:

<center>Centered text</center>
Added: HTML3.2 | Removed: HTML5

<font>


Why removed: Entirely presentational, CSS is the standard.

Old way:

<font color="red" face="Verdana">Old style</font>
Added: HTML3.2 | Removed: HTML5

<strike>


Why removed: Use <del> for deleted text or CSS text-decoration.

Old way:

<strike>Strikethrough</strike>
Added: HTML2 | Removed: HTML5

<tt>


Why removed: Presentational monospace. Use <code> or CSS.

Old way:

<tt>Monospaced text</tt>

MODERN REPLACEMENTS (OLD → NEW)

These tags still work but are deprecated. Use modern alternatives.

<b> (Bold)

Makes text bold (presentational only).


Modern replacement: <b> → <strong>

OUT PUT:

<b>Bold text</b> (visual only)
<strong>Important text</strong> (semantic)

<i> (Italic)

Makes text italic (presentational).


Modern replacement: <i> → <em>

<i>Italic text</i>
<em>Emphasized text</em>

<u> (Underline)

Underlines text – causes confusion with links.


Modern replacement: CSS <u> → CSS
<menu> (old version)

List of menu items, identical to <ul>.

✅ Modern replacement: <ul> or <nav> with list items.
<dir> (Directory list)

Created directory listing, same as <ul>.

✅ Modern replacement: <ul>
<pre> (Preformatted)

Still used but the 'width' attribute is obsolete.

✅ Modern replacement: <pre> with CSS styling.
Various Table Attributes

align, bgcolor, border, cellpadding, cellspacing, valign, width.

✅ Modern replacement: CSS (text-align, background-color, border, padding, etc.)
Image Attributes (align, border, hspace, vspace)
✅ Modern replacement: CSS (float, border, margin)
Body Attributes (bgcolor, text, link, vlink, alink)
✅ Modern replacement: CSS on <body> element.

📜 Fun Facts About Old HTML Tags

  • 🔴 The <blink> tag was created by Netscape and called "the most annoying tag ever"
  • 🔴 <marquee> was never an official HTML standard - only Internet Explorer!
  • 🔴 <plaintext> was dangerous because everything after it became unrenderable text
  • 🔴 <xmp> and <listing> were the original code display tags before <pre>
  • 🔴 Framesets were hated because they broke bookmarks and SEO
  • 🔴 <applet> died because Java plugins became a security nightmare
  • 🔴 The <font> tag was the #1 reason why CSS was invented!
